Step 11: Import `c02games.csv` & Set Primary Key

Our first task is to import the main data file and set it up correctly. The primary key is a unique identifier for each record in a table. By examining c02games.csv, we can see that Game_Code is unique for every game, making it the perfect primary key.

Importing and Setup Process:

  1. Open MS Access and create a new, blank database.
  2. Go to the External Data tab, click New Data Source → From File → Text File.
  3. Browse for and select c02games.csv.
  4. In the wizard, ensure Delimited is selected and check the box for First Row Contains Field Names. The delimiter should be a comma.
  5. On the next screen, you can set the data types. Click the Advanced... button for more control. Set the data types as specified in the exam paper. Pay close attention to numbers, currency, and decimals.

Step 14: Create One-to-Many Relationships

Linking tables allows us to combine data from all three sources in our reports. We will connect the tables using their common fields (the primary keys we just set).

  1. Go to the Database Tools tab and click on Relationships.
  2. Add all three tables to the Relationships window.
  3. Click and drag Gen_Code from the genre table and drop it onto Gen_Code in the games table. Click "Create" in the pop-up window.
  4. Click and drag Age_Code from the classifications table and drop it onto Age_Code in the games table. Click "Create".

Step 16: Produce the 'Latest Cyrus Console Games' Report

For complex reports, it's best to create a query first to select and filter the data, and then use that query to build the report.

Part A: Create the Query

  1. Go to Create → Query Wizard → Simple Query Wizard.
  2. Add the required fields from all three tables: Game_Title, Genre, Age_Category, Release_Year, Console_Type, and Rank.
  3. Save the query and open it in Design View.
  4. In the "Criteria" row, enter:
    • Under Console_Type: Like "Cyrus*" (The * is a wildcard).
    • Under Release_Year: >=2010.
  5. In the "Sort" row, select Ascending under the Rank field.
  6. Run the query to test it. Save and close.

Part B: Create the Report

  1. Go to Create → Report Wizard.
  2. Select your query as the data source. Add all the fields.
  3. Do not add any grouping levels. The data should already be sorted by rank.
  4. Choose a Tabular layout and Portrait orientation.
  5. Title the report Latest Cyrus Console Games and open it in Design or Layout View to format it.
  6. Make the title larger, ensure all data and labels are fully visible, and check in Print Preview that it fits on a single page.
  7. Finally, add your name, centre number, and candidate number to the report header or footer.

Step 17: Produce the 'Junior Tomas Games' Report

This report is more advanced, involving a run-time calculation and a summary total.

Part A: Create the Query

  1. Go to Create → Query Design and add the games and classifications tables.
  2. Add the required fields.
  3. In the "Criteria" row, enter:
    • Under Game_Title: Like "*Tomas*".
    • Under Age_Code: "C3" Or "C7" (you may need to add this field just for the criteria, then uncheck 'Show').
  4. Create the calculated field: In an empty field column, type: Sales_%: ([Global_Sales]/354)*100. Right-click this new field, go to Properties, and set the Format to Standard and Decimal Places to 2.
  5. Set the sort order: Ascending for Console_Type and Descending for Global_Sales.

Part B: Create and Format the Report

  1. Use the Report Wizard with your new query as the source.
  2. Select a Tabular layout and Landscape orientation.
  3. Title the report Junior Tomas Games.
  4. Open in Layout or Design View. Resize the title and columns to fit a single page width.
  5. Calculate the average: Click on any value in the Price column. Go to the Report Design tab, click Totals, and select Average.
  6. Add a new label to the left of the calculated average with the text Average game price.
  7. Add your name, centre number, and candidate number in the footer.

Evidence Requirement

In the report's Design View, click the calculated average field. In the Property Sheet, you will see its Control Source is =Avg([Price]). Take a screenshot showing this formula and paste it into your evidence document.
