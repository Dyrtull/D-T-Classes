A flowchart is a diagram that shows the steps in an algorithm using standard symbols connected by arrows to show the flow of control.

What is Pseudocode?

Pseudocode uses English-like statements to describe algorithm steps. It's more detailed than flowcharts but easier to read than actual programming code.

Key Pseudocode Keywords

Variables & Input/Output
  • DECLARE variable : INTEGER
  • INPUT variable
  • OUTPUT "message", variable
  • variable ← value (assignment)
Selection
  • IF condition THEN
  • ELSE
  • ENDIF
  • CASE OF variable
Iteration
  • FOR counter ← 1 TO 10
  • WHILE condition DO
  • REPEAT...UNTIL condition
  • NEXT counter
Operators
  • =, >, <, >=, <=, <>
  • AND, OR, NOT
  • +, -, *, /, MOD, DIV

Same Example in Pseudocode

DECLARE Number : INTEGER
INPUT Number
IF Number > 0
THEN
    OUTPUT "Positive"
ELSE
    IF Number < 0
    THEN
        OUTPUT "Negative"
    ELSE
        OUTPUT "Zero"
    ENDIF
ENDIF

Enhanced Concepts and Examples

Bubble Sort

This algorithm repeatedly steps through the list, compares adjacent elements, and swaps them if they are in the wrong order. The pass through the list is repeated until the list is sorted. It is a simple but inefficient sorting algorithm.

// Bubble Sort Pseudocode
PROCEDURE bubbleSort(list)
    DECLARE listSize : INTEGER
    DECLARE swapped : BOOLEAN
    listSize ← LENGTH(list)
    
    REPEAT
        swapped ← FALSE
        FOR i ← 0 TO listSize - 2
            IF list[i] > list[i+1] THEN
                temp ← list[i]
                list[i] ← list[i+1]
                list[i+1] ← temp
                swapped ← TRUE
            ENDIF
        NEXT i
    UNTIL NOT swapped
ENDPROCEDURE

Binary Search

A much more efficient search algorithm than linear search, but it requires the data to be sorted first. It works by repeatedly dividing the search interval in half. It's perfect for searching large, sorted datasets.

// Binary Search Pseudocode
FUNCTION binarySearch(list, item) RETURNS INTEGER
    DECLARE low, high, mid : INTEGER
    DECLARE found : BOOLEAN
    low ← 0
    high ← LENGTH(list) - 1
    found ← FALSE
    
    WHILE low <= high AND found = FALSE DO
        mid ← (low + high) DIV 2
        IF list[mid] = item THEN
            found ← TRUE
            RETURN mid
        ELSE IF list[mid] < item THEN
            low ← mid + 1
        ELSE
            high ← mid - 1
        ENDIF
    ENDWHILE
    
    RETURN -1 // Not found
ENDFUNCTION

Check Digits

A check digit is a form of validation that is calculated from the other digits in a number. It is used to detect errors, such as typing a number incorrectly. A common method is the Modulo 11 algorithm.

To validate, you re-calculate the check digit from the main number and compare it to the given check digit. If they match, the number is considered valid. Example: ISBN numbers use a check digit.

New Concepts and Examples

Procedures vs. Functions

The key difference is that a function must always return a value, while a procedure does not. Functions are used for calculations or retrieving data, while procedures are used to perform a set of actions, such as printing a report or updating a record.

// Example of a FUNCTION
FUNCTION calculateVAT(price : REAL) RETURNS REAL
    // Calculates VAT at 20%
    RETURN price * 0.2
ENDFUNCTION

// Example of a PROCEDURE
PROCEDURE printReport(name : STRING)
    OUTPUT "Report for:", name
ENDPROCEDURE

Local vs. Global Variables

A global variable is declared at the start of the program and can be accessed and modified from anywhere. A local variable is declared inside a procedure or function and can only be used within that specific block of code.

Examiner Tip: Using global variables can make code difficult to debug, as their value can change unexpectedly. It is best practice to use local variables and pass data using parameters. 

DECLARE globalVariable : INTEGER
globalVariable ← 10

  3. Loops:
    • Pre-condition loop: The condition is checked before the loop body is executed. This means the loop might not run at all if the condition is initially false. Example: `WHILE...DO...ENDWHILE` or `FOR...TO...NEXT`. Use case: Input validation where you need to check if the user's input is valid before processing it.
    • Post-condition loop: The condition is checked after the loop body is executed. This means the loop will always run at least once, regardless of the condition. Example: `REPEAT...UNTIL`. Use case: A login screen where you must get the user's input at least once before checking if it is correct.

Library Routines

These are built-in functions that perform common tasks.

  • MOD (Modulo): Returns the remainder of a division. Example: `10 MOD 3` returns `1`.
  • DIV (Integer Division): Returns the whole number part of a division. Example: `10 DIV 3` returns `3`.
  • ROUND: Rounds a real number to the nearest integer. Example: `ROUND(3.7)` returns `4`.
  • RANDOM: Generates a random number. Example: `RANDOM(1, 10)` generates a random integer between 1 and 10 inclusive.

File Handling Operations

These operations are used to read from and write to text files for permanent data storage.

  • OPENFILE("filename", mode): Opens a file. `mode` can be `READ`, `WRITE`, or `APPEND`.
  • READFILE("filename"): Reads a single line from the file.
  • WRITEFILE("filename", data): Writes data to the file.
  • CLOSEFILE("filename"): Closes the file, saving any changes.

Concept Definitions

  • Database: A structured collection of data that allows for the retrieval of information.
  • Field: A column in a database table that stores a single piece of data for each record, like "StudentName" or "Age".
  • Record: A row in a database table containing all the information about a single entity, such as a student or a product.
  • Primary Key: A field in a database table that uniquely identifies each record.
  • SQL (Structured Query Language): The standard language for querying a database to retrieve or manipulate data.
  • Logic Gate: An electronic circuit that takes one or more binary inputs and produces a single binary output.
  • Truth Table: A table that lists all possible combinations of binary inputs to a logic circuit and the resulting output.
