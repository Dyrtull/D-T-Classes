How IoT Web Servers Work

The ESP32 is a Server Basic

A "Server" is just a device that waits for requests. When your phone (the Client) asks for the "home page", the ESP32 sends a block of HTML text.

The IP Address Intermediate

Every device on WiFi has a unique address (like 192.168.1.15). You type this into your browser to find your feeder.

WiFi Stability & Power Advanced

WiFi communication uses radio bursts that pull high current. If your power supply is weak, the WiFi will drop. The expansion board's power rails are vital here!

CCQ: "Can you access the feeder if your phone is on Cellular data and the ESP32 is on home WiFi?"

Not by default. Both must be on the same local network because the 192.168.x.x address is "private" to your house router.

Building the Base Code

This code transforms your ESP32 into a simple website host.

#include <WiFi.h>
#include <WebServer.h>

WebServer server(80); // Standard HTTP port

void handleRoot() {
  String html = "<h1>Pet Feeder</h1>";
  html += "<a href='/feed'><button>FEED</button></a>";
  server.send(200, "text/html", html);
}

void setup() {
  WiFi.begin("YourSSID", "Password");
  server.on("/", handleRoot);
  server.begin();
}
