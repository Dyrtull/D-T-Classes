Step 21: Import Students Table

First, you need to import the student data and set it up correctly in your database.

  1. Open your database package and import `J218STUDENTS.csv`.
  2. Use the following field names and data types:
    • First_Name, Family_Name, Student_ID, Faculty, Telephone: Text
    • Birth_Date: Date (format as dd-MMM-yy)
    • Full_Time: Boolean/Logical (display as Yes/No)
  3. Set the Student_ID field as the primary key.
  4. Save the table. You can skip taking screenshot evidence.

Step 22: Import Test Data & Create Relationship

Next, import the test results and link the two tables together.

  1. Import the file `J218TEST_DATA.csv` as a new table.
  2. Ensure date and time fields are imported with the correct formats.
  3. Do not set a primary key for this table.
  4. Create a one-to-many relationship between the two tables.
  5. Link `Student_ID` from the students table to `Reg_ID` from the test data table.
  6. Enforce referential integrity.
  7. Save the relationship. You can skip taking screenshot evidence.

Step 23: Create Data Entry Form

Build a simple form to make adding new test data easy and consistent.

  1. Create a new form based on the test data table.
  2. Include all fields from the table.
  3. Choose a columnar layout. This will stack the fields vertically.
  4. Ensure all field headings are aligned to the left.
  5. Adjust the layout so that all field headings and their data are fully visible.

Step 27: Data Validation Concepts

This section covers key data validation concepts mentioned in the task.

a) Definition of Data Validation

Data validation is an automatic check performed by a computer to ensure that data entered is sensible and reasonable. It helps prevent incorrect data from being entered, but it does not guarantee that the data is 100% accurate.

b) Validation for 'Score' Field

The most appropriate type of validation would be a Range Check.

c) Validation Criteria for 'Score'

Since the test is marked out of 60, the criteria would be to accept values that are greater than or equal to 0 AND less than or equal to 60.

d) Another Validation Type

Other common types of validation include: Presence Check (ensures a field is not left empty), Format Check (ensures data matches a pattern, like a postcode), or Type Check (ensures data is of the correct type, e.g., numeric).
