Core Concepts

Input devices are hardware components that allow users to interact with a computer system. They enable the user to input data or commands into the system, which the computer then processes to produce an output .

Touch Screen Types

  • Resistive Touch Screen: Detects touch by pressing a flexible screen layer. Used in ATMs and older smartphones for direct interaction
  • Infra-red Touch Screen: Detects touch by interrupting infrared light beams across the screen. Used in kiosks and large touch displays with accurate detection
  • Capacitive Touch Screen: Detects touch through changes in electrical charge on screen surface. Used in modern smartphones and tablets for smooth, responsive multi-touch experience

Scanner Types

  • Two-Dimensional (2D) Scanner: Captures flat, 2D images of documents or photos. Used in offices, libraries, schools to digitise documents and photos for storage, editing, or sharing
  • Three-Dimensional (3D) Scanner: Captures detailed 3D shapes and dimensions of objects. Used in manufacturing, design, medical imaging to create digital models for design, analysis, or reproduction

Output devices are hardware components that receive information from a computer system and present it to the user in an understandable form. They enable the computer to communicate the results of processed data or commands .

Choosing the Right Device

When recommending a device for a specific situation, consider these factors :

  • User Needs: What tasks will the user perform? (e.g., graphic designer needs graphics tablet, data entry clerk needs keyboard with number pad)
  • User Skills: Is the user comfortable with the device? (touch screen vs mouse/keyboard preference)
  • Environment: Where will it be used? (wireless mouse for clutter-free office vs wired mouse for public lab to prevent theft)
  • Cost: Higher-end devices have more features but cost more - consider budget and value of extra features

Sensors are input devices that measure a physical property of their environment such as light levels, temperature or movement. Sensors can be used for monitoring systems and control systems .

System Types

  • Monitoring System: Tracks the state of a system, gathers data and may issue warning messages
  • Control System: Controls the system based on input from sensors (e.g., fish tank heater that automatically starts when water temperature falls below acceptable level)
  • Feedback Loop: A process where outputs are recycled and used as inputs, creating a continuous cycle

Primary storage is directly accessed by the CPU. Computer systems need both primary and secondary storage to operate .

Why Primary Storage is Needed

A computer needs primary storage because access times are considerably faster than secondary storage. This dramatically reduces the time taken to complete operations such as the Fetch-Execute Cycle .

  • Holds data and instructions that the CPU needs to access whilst computer is turned on
  • Used as short-term, working memory in hardware directly connected to CPU
  • Higher performance means much higher cost, limiting the amount used
  • RAM commonly purchased in 16-32 GB whereas secondary storage is in terabytes

RAM (Random Access Memory)

RAM is primary storage directly connected to the CPU and holds data and instructions currently in use .

  • Volatile: Contents lost when power is turned off
  • Very fast: Much faster than secondary storage
  • Read/write: Data can be read from and written to
  • Larger capacity: Compared to ROM
  • CPU Access: Data and instructions must be copied from secondary storage to RAM for CPU access

ROM (Read Only Memory)

ROM is primary storage that holds the first instructions a computer needs to start up (Bootstrap) .

  • Contains BIOS: Basic Input Output System
  • Location: Small memory chip on the motherboard
  • Non-volatile: Contents not lost when power is turned off
  • Read only: Data can only be read from
  • Speed: Fast memory, faster than secondary storage but slower than RAM
  • Smaller capacity: Compared to RAM
Primary Storage Secondary Storage
Volatile (except ROM) Non-volatile
Small capacity Large capacity
Very fast access times Slower access times

Secondary storage devices are non-volatile storage that retain digital data within a computer system. They provide means of storing, accessing, and retrieving data including software applications, documents, images, videos, and more .

Why Secondary Storage is Needed

  • Long term storage of programs and data currently not in use
  • ROM is read only and RAM is volatile, so secondary storage needed
  • Holds programs and data whilst computer is turned off (non-volatile)
  • Slower than primary storage but much higher capacity - perfect for backup and archive

Storage Characteristics

  • Capacity: Maximum amount of data that can be stored
  • Speed: How fast data can be read from and written to
  • Cost: Price consideration
  • Portability: How easy to move around, physical size and weight
  • Durability: How robust is the storage
  • Reliability: Probability device will consistently perform its function

Magnetic Storage

Magnetic storage uses magnets (polarity) to store binary 0s and 1s. Made up of several metal discs (platters) coated with magnetic material where iron particles are magnetised to represent data .

Advantages
  • High storage capacity
  • Low cost per gigabyte
  • Moderate read/write access speed
Disadvantages
  • Moving parts can get damaged if dropped
  • Heavy and bulky, less convenient for transport
  • Prone to mechanical failure
  • Loud (spinning disks)

Solid-State (Flash Memory)

Solid-state storage uses electronic circuits to store binary 0s and 1s. Examples include SSDs and USB flash drives. Uses NAND and NOR gates in electrical circuits with cells containing transistors that act as switches .

Advantages
  • Medium/high storage capacity
  • Very fast read/write access
  • No moving parts (durable)
  • Small and portable
  • Silent operation
Disadvantages
  • Very high cost per gigabyte
  • Limited read/write cycles

Optical Storage

Optical storage uses lasers to burn the surface of a disk, creating pits and lands for storing binary 0s and 1s. Examples include CDs, DVDs, and Blu-rays. Works by shining laser at disk and processing the reflection .

Advantages
  • Very low cost per gigabyte
  • No moving parts (durable)
  • Small and portable
  • Silent operation
Disadvantages
  • Very low capacity
  • Very slow read/write access
  • Prone to scratches

Virtual Memory

Virtual memory is an extension of primary storage (RAM) located on secondary storage. Used when RAM is close to being full so computer remains operational .

  • Programs and data not currently being executed are transferred to virtual memory
  • Programs stored as pages of data, transferred between RAM and virtual memory when needed
  • Much slower than RAM and negatively impacts system performance
  • To avoid virtual memory use, increase RAM size

Cloud Storage

Cloud storage describes long-term secondary storage of data in a remote location, accessible only via wide area network (Internet). Data stored on remote servers using magnetic (HDD) or solid state (SSD) storage .

Advantages
  • Access data anywhere, any time, from any device
  • Automatic backups
  • Endless storage space (cost permitting)
  • Multiple people can access same file simultaneously
  • Advanced security features (encryption, multi-factor authentication)
  • Scalable - increase or decrease capacity as needed
Disadvantages
  • Monthly/yearly subscription cost
  • No access without internet connection
  • Can be expensive for large amounts of data
  • Data sent over internet can be intercepted
  • User doesn't have complete control over security

Network hardware is a selection of essential components that enable connectivity and communication of devices within computer networks [file:4].

Network Interface Card (NIC)

The Network Interface Card (NIC) is required for a computer to connect to a network. A NIC can be both wired and wireless and allows your computer to send and receive data over a network [file:4].

Address Systems

Devices on a network send and receive data. A device needs an address to ensure it sends data to the correct place. There are two types of network address systems [file:4]:

MAC Address

A MAC (Media Access Control) address is a unique identifier for devices on local area networks (LAN) [file:4].

  • Given to NIC at point of manufacture [file:4]
  • Static - can never change [file:4]
  • Stays same if device moves to different network [file:4]
  • 12 hexadecimal digits (48 bits), grouped in pairs [file:4]
  • First 3 pairs: manufacturer ID (OUI) [file:4]
  • Last 3 pairs: serial number of NIC [file:4]
  • Enough addresses for 281 trillion devices [file:4]
IP Address

An IP (Internet Protocol) address is a unique identifier for devices on the Internet (WAN) [file:4].

  • Can be static (stays same) or dynamic (can change) [file:4]
  • Static used for servers and websites [file:4]
  • Dynamic assigned by DHCP server [file:4]
  • Changes if device moves to different network [file:4]
  • IPv4: 4 blocks (0-255), 32 bits, 4+ billion addresses [file:4]
  • IPv6: 8 blocks of 4 hex digits, 128 bits, billions of addresses per person [file:4]

Key Differences: MAC vs IP Addresses

MAC Address IP Address
Static - cannot change [file:4] Dynamic - can change [file:4]
Used for LAN communication [file:4] Used for WAN/Internet communication [file:4]

Router

The router is responsible for routing data packets between different networks. It connects local area networks (LAN) to the wider internet (WAN) [file:4].

Router Tasks:
  • Send and receive packets of data [file:4]
  • Connect local network to the internet [file:4]
  • Assign IP addresses to devices on the network [file:4]
  • Convert packets from one protocol to another [file:4]
  • Manage and prioritise data traffic for stable connections [file:4]
  • Direct internet traffic to correct destination/devices [file:4]

Key Vocabulary

Input Device

Hardware that allows users to enter data into a computer system.

Output Device

Hardware that presents processed data to users in a readable form.

Primary Storage

Memory directly accessible by the CPU, typically volatile and fast.

Secondary Storage

Non-volatile storage for long-term data retention, slower than primary storage.

MAC Address

Unique identifier for network interface cards on local networks.

IP Address

Unique identifier for devices on wide area networks or the internet.
