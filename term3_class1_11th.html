📚 The Architect's Cheat Sheet

Before we touch the live servers, you need to understand the two core philosophies of data architecture.

Flat-File vs. Relational Database

Architecture Flat-File Database Relational Database
Structure One single table containing all data. Multiple linked tables.
Data Redundancy High — data is repeated across rows. Low — data stored once, referenced via keys.
Example A single spreadsheet of all citizens + their districts + energy bills in one row. Separate CITIZENS, DISTRICTS, and ENERGY_BILLS tables, linked by ID.
Risk Update anomaly — change one address, miss 50 duplicate rows. Requires careful key design.

🎯 Exam-Precision Definition: A flat-file database stores all data in a single table, leading to data redundancy. A relational database uses multiple tables linked by key fields to eliminate redundancy and improve data integrity.

The Data Type Arsenal (0417 Taxonomy)

Data types are contracts between the city and the database engine. Break the contract, and the system crashes.

Data Type What It Stores Medellín 2050 Example Key Rule
Text (Alphanumeric) Letters, numbers, symbols — but NOT for calculation. CitizenID, Surname Use when you will NEVER calculate with the value.
Integer (Numeric) Whole numbers only. EnergyCredits, FloorNumber No decimal places.
Decimal (Numeric) Numbers with decimal places. TemperatureSensor Specify decimal places.
Currency Monetary values. FineAmount Prevents rounding errors, auto-adds symbol.
Date/Time Dates and/or times. DateOfBirth Must set display format (e.g., dd/mm/yyyy).
Boolean / Logical Two-state only: Yes/No, True/False. IsVerified Eliminates invalid entries like "maybe".
⚠️ Common Error: Using Integer for a field like CitizenID (e.g. C-00451). This fails because the hyphen makes it non-numeric. IDs with letters or symbols are always Text.

The Primary Key: The City's Fingerprint

🎯 Exam-Precision Definition: A primary key is a field (or combination of fields) that uniquely identifies each record in a table. It cannot be null and cannot contain duplicate values.

The Analogy: Every citizen of Medellín 2050 has a chip implanted at birth with a unique 7-character ID. No two chips are identical. Even twins. That chip ID is the primary key.

  • ✅ Unique: No duplicates allowed.
  • ✅ Never empty: No nulls allowed.
  • ✅ Stable: It should never need to change.
  • ✅ Minimal: As short as possible while remaining unique.

MS Access Implementation Steps

  1. Create new database: Open Microsoft Access, select Blank Desktop Database, and name it Medellin2050_SmartCity.accdb.
  2. Design View: Right-click on 'Table1', choose Design View, and save the table as CITIZENS. Define the field structure exactly as follows:
    • CitizenID -> Short Text (Contains a hyphen).
    • DateOfBirth -> Date/Time (Use Field Properties to set format to Short Date: dd/mm/yyyy).
    • EnergyCredits -> Number (Set Field Size to Integer in properties - no fractions of credit).
    • FineAmount -> Currency (Handles the monetary symbol automatically).
    • IsVerified -> Yes/No (Boolean logic).
  3. Primary Key: Select the CitizenID row and click the 🔑 Primary Key button in the top ribbon. This is the city's fingerprint!
  4. The Deliberate Error: Temporarily change EnergyCredits to Short Text. Switch to Datasheet View, save, and type "plenty" in the field. When you switch back to Number in Design View, Access warns of data loss. The type is the law. Design schema BEFORE importing data.
  5. Import the CSV: Go to the External Data tab > New Data Source > From File > Text File. Select your downloaded CSV, choose to Append a copy of the records to the table CITIZENS, and finish the wizard.

📖 Digital Glossary

Flat-File Database
A database structure that stores all data in a single, large table. Prone to data redundancy and update anomalies.
Relational Database
A database structure that uses multiple tables linked by key fields to eliminate redundancy and improve data integrity.
Primary Key
A field (or combination of fields) that uniquely identifies each record in a table. It cannot be null and cannot contain duplicate values.
Data Redundancy
The unnecessary repetition of data in a database, which wastes space and increases the chance of errors when updating.
Text / Alphanumeric (Data Type)
Stores letters, numbers, and symbols. Used for data that will not be part of mathematical calculations (e.g., Phone numbers, IDs).
Integer (Data Type)
Stores whole numbers only. No decimal places.
Boolean / Logical (Data Type)
Stores data that can only have one of two possible states (e.g., Yes/No, True/False, On/Off).
