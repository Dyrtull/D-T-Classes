Understanding Data Transmission

A guide to how information moves across networks.

Data Transmission & Packets

Data transmission is the process of moving data from one point to another. To do this efficiently and reliably, data is often broken down into smaller units called data packets.

Packet Structure

  • Header: Contains the IP address of the sender and the receiver, the sequence number, and the size of the packet.
  • Payload: Contains the actual data being transmitted.
  • Trailer: Includes a method of identifying the end of the packet and error-checking information.

Packet Switching

Packet switching is a method where data is broken into multiple packets. These packets are sent independently from start to end and reassembled at the receiverâ€™s computer.

Advantages

  • No need to create a single line of communication.
  • Possible to overcome failed or busy nodes.
  • High data transmission speed.
  • Easy to expand package usage.

Disadvantages

  • Packets may be lost.
  • More prone to errors in real-time streaming.
  • Delay at the receiver while packets are being re-ordered.

Types of Data Transmission

Simplex: Data transmission is in one direction only (e.g. computer to printer).

Half-duplex: Data transmission is in both directions but not at the same time (e.g., walkie-talkie).

Full-duplex: Data transmission is in both directions simultaneously (e.g. broadband connection on a phone line).

Universal Serial Bus (USB)

USB is an asynchronous serial data transmission method using a four-wire shielded cable (two for power/earth, two for data).

Advantages

  • Automatically detected.
  • Only fits one way, preventing incorrect connections.
  • Backwards compatible.
  • Industry-standard.

Disadvantages

  • Transmission rate is less than 120 MB/sec.
  • Maximum cable length is about 5 metres.
  • Different data transmission rates.

Methods of Error Detection

1. Parity Checks

Uses the number of 1-bits in a byte to check for errors. The LMB (Left-Most Bit) is the parity bit.

2. Checksum

The sender calculates a checksum value from a block of data. The receiver performs the same calculation and compares the result. If they don't match, the data is re-sent.

3. Echo Check

The receiver sends the data back to the sender for verification. The sender compares the received data with the original to check for errors.

Other Important Methods

Check Digits

Calculated from all other digits in a code (e.g., product barcodes) to detect mistyping errors.

Example: A common ISBN check digit system can detect if "6372" was typed incorrectly as "6379".

Automatic Repeat Requests (ARQs)

Uses acknowledgements (ACK) and timeouts to ensure data is received correctly. If a negative acknowledgement (NACK) or no response is received, the data is re-sent.

Encryption

The process of turning data into an unreadable ciphertext from its original plaintext form to prevent unauthorized access.

Symmetric Encryption

Uses the same key for both encryption and decryption.

Asymmetric Encryption

Uses a public key for encryption and a separate private key for decryption.
