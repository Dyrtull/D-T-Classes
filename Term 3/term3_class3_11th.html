📚 The City Planner's Cheat Sheet

Without relationships, your tables are just isolated spreadsheets that happen to live in the same file. Relationships are what make a database relational.

The Core Vocabulary (0417 Precision)

Term Formal IGCSE Definition Medellín 2050 Analogy
Primary Key A field that uniquely identifies each record in its own table; cannot be null or duplicated. The citizen's biometric chip ID.
Foreign Key A field in one table that references the primary key of another table, creating a link between them. The CitizenID field inside ENERGY_BILLS — it points back to a real person.
Relationship A defined link between two tables based on matching key fields. The bridge connecting the Citizens Registry to the Finance Department.
Referential Integrity A rule preventing a foreign key value from existing without a matching primary key value in the related table. The city law: "You cannot bill a citizen who doesn't exist."

Relationship Types: The Three City Structures

The syllabus expects you to recognise all three, even though you mainly build One-to-Many.

Type Description City Example
One-to-One One record in A matches exactly one record in B. One citizen ↔ one biometric profile.
One-to-Many One record in A can be linked to many records in B. One district ↔ many citizens.
Many-to-Many Many in A match many in B. Many citizens ↔ many transport routes. (Requires a junction table).
🎯 Exam Trap: Do not write "two-to-many" or "multiple-to-one." The correct terms are strictly one-to-one, one-to-many, and many-to-many. Marks are lost for informal descriptions.

Referential Integrity: The Two Cascade Options

When you enforce referential integrity in Access, you get two optional cascade rules:

  • Cascade Update Related Fields: If a Primary Key value changes in Table A, all matching Foreign Key values in Table B update automatically. (Safe: Use when PK values might legitimately change).
  • Cascade Delete Related Records: If a record is deleted from Table A, all linked records in Table B are deleted automatically. (Dangerous: Deleting a district deletes all citizens in it!).

⚠️ Teaching point: In most exam scenarios, cascade delete is dangerous. The safer answer on Paper 1 is: "Referential integrity should prevent deletion of a parent record while child records still exist."

Step 3: The ENERGY_BILLS Table

Create a new table called ENERGY_BILLS. Set BillID as AutoNumber (Primary Key 🔑). Ensure it has a CitizenID field (Short Text) acting as its Foreign Key. Import the CSV.

Step 4: Build the Relationships in Access

  1. Go to Database Tools → Relationships.
  2. Add all three tables to the window.
  3. Relationship 1 (DISTRICTS → CITIZENS): Drag DistrictCode from DISTRICTS onto DistrictCode in CITIZENS. Check Enforce Referential Integrity and Cascade Update. Click Create. (You should see a 1 to ∞ line).
  4. Relationship 2 (CITIZENS → ENERGY_BILLS): Drag CitizenID from CITIZENS onto CitizenID in ENERGY_BILLS. Check Enforce Referential Integrity and Cascade Update. Click Create.

The Breaking Test: Go to ENERGY_BILLS in Datasheet View. Try to add a new record with CitizenID = C-00999. Access blocks it because C-00999 doesn't exist in the CITIZENS table. You have successfully prevented orphan records!

📖 Digital Glossary

Primary Key
A field that uniquely identifies each record in its own table; cannot be null or duplicated.
Foreign Key
A field in one table that references the primary key of another table, creating a link between them.
Relationship
A defined link between two tables based on matching key fields.
Referential Integrity
A rule that prevents a foreign key value from existing without a matching primary key value in the related table.
One-to-Many
One record in Table A can be linked to many records in Table B, but each record in B links to only one record in A.
Orphan Record
A record in a child table (like ENERGY_BILLS) whose foreign key points to a primary key that no longer exists in the parent table.
Cascade Update
A referential integrity option: If a Primary Key value changes, all matching Foreign Key values update automatically.
Cascade Delete
A referential integrity option: If a record is deleted from the parent table, all linked records in the child table are deleted automatically.
