📚 The Intelligence Analyst's Cheat Sheet

What is a Query? (0417 Precision)

🎯 Exam-Precision Definition: A query is a search of a database that extracts records meeting specified criteria, and can display selected fields from one or more tables in a specified order.

Feature Table Query
Stores data? ✅ Yes — permanently ❌ No — shows a live view of data
Updates when data changes? N/A ✅ Yes — re-runs against current data
Can combine tables? No ✅ Yes — via JOIN relationships
Can calculate new values? No ✅ Yes — via Calculated Fields

The Criteria Arsenal

This is the vocabulary of database intelligence. You must know these for Paper 2.

Operator Syntax in Access QBE What it Does
Equals "El Poblado" Exact text match.
Not Equals <>"El Poblado" Excludes the exact match.
Greater/Less >300 or <100 Numeric comparison.
Between Between 100 And 500 Inclusive numeric range.
Wildcard (Many) Like "C-001*" * matches any number of characters.
Wildcard (One) Like "SN-00?" ? matches exactly one character.
Null Check Is Null or Is Not Null Finds empty or non-empty fields.
Date Range >=#01/02/2050# Uses hashes # around dates.

1. Calculated Fields

A calculated field does not exist in the table — the database computes it on the fly each time the query runs. The syntax is:

NewFieldName: [ExistingField1] operator [ExistingField2]

Example: SurchargeTotal: [AmountDue]*1.1

⚠️ Critical Rule: You must use square brackets [ ] around existing field names so Access knows they are fields, not text.

2. The AND vs. OR Trap (Crucial for Paper 2)

  • AND: Both conditions must be true simultaneously. In Access, put criteria on the same row.
  • OR: Either condition can be true. In Access, put criteria on different rows (the "or:" row).
⚠️ The Trap: If you want "citizens in El Poblado and Laureles", and you write "El Poblado" AND "Laureles" in the same row, it means a citizen must live in BOTH sectors at the exact same time. Impossible. They must be on separate OR rows.

📖 Digital Glossary

Query
A search of a database that extracts records meeting specified criteria, and can display selected fields from one or more tables.
Criteria
The specific conditions or rules that data must match to be included in the query results.
Calculated Field
A field in a query that computes new values (using math or logic) based on existing data, created on the fly and not stored in the table.
Wildcard (*)
A symbol that substitutes for any number of unknown characters in a text search (e.g., Like "R*").
Wildcard (?)
A symbol that substitutes for exactly one unknown character in a text search (e.g., Like "M????").
AND Logic
A condition where multiple criteria must ALL be true simultaneously for a record to be selected.
OR Logic
A condition where at least ONE of the criteria must be true for a record to be selected.
Parameter Query
A query that prompts the user for input (criteria) via a dialog box every time it is run (e.g., >[Enter minimum amount:]).
