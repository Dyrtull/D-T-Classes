📚 The Security Architect's Cheat Sheet

A database without validation is like a city with no ID checkpoints — anyone and anything gets in. The purpose of validation is to enforce data integrity at the point of entry.

The Six Validation Types (0417 Taxonomy)

The syllabus explicitly names all six of these. You must know the name, definition, an example, and one limitation.

Validation Type Formal Definition Medellín 2050 Example
Presence Check Ensures a field is not left empty (null). CitizenID must not be blank.
Range Check Ensures a numeric or date value falls within an acceptable minimum and maximum. EnergyCredits must be between 0 and 9999.
Length Check Ensures the number of characters in a field is within specified limits. CitizenID must be exactly 7 characters (e.g., C-00101).
Format Check Ensures data matches a specific pattern of characters. CitizenID must match pattern L-NNNNN (Letter, hyphen, 5 digits).
Type Check Ensures the data entered is of the correct data type. EnergyCredits must be a number (rejects text like "plenty").
Lookup Check Ensures a value entered matches an item in a predefined list. Sector must be one of: El Poblado, Laureles, Envigado, Belén, Centro.

Validation in MS Access: Three Mechanisms

There are distinct tools in Access to enforce validation. Do not confuse them.

Mechanism Where Set What It Does Example
Data Type Field Data Type column Rejects data of the wrong type entirely. (This acts as your Type Check). Setting EnergyCredits as Number rejects text.
Validation Rule & Text Field Properties Custom logical expression data must satisfy. Text is the error message shown to the user. Rule: >=0 And <=9999
Text: "Credits must be between 0 and 9999"
Input Mask Field Properties Forces a specific character-by-character entry pattern (Combines Format & Length checks). L\-00000 enforces C-00101 format.
Lookup Wizard Data Type dropdown Creates a dropdown list of permitted values. Sector dropdown with 5 city zones.

The Core Philosophy & The "Accuracy" Trap

🎯 Exam-Precision Statement: "Validation is an automatic check carried out by the computer to ensure that data entered is reasonable, complete, and within acceptable boundaries. It cannot verify that data is accurate or true."

⚠️ The Classic Paper 1 Trap

A range check on DateOfBirth between 01/01/2010 and 04/03/2026 would successfully reject 99/99/9999.
But it would accept 15/05/2025 even if the real citizen was actually born in 1985.

Validation cannot detect lies, only nonsense. Checking truth is called Verification (a manual check).

🛠️ The Live Build: Hardening the CITIZENS Table

Open Medellin2050_SmartCity.accdb and switch to Design View for the CITIZENS table.

MS Access Expression Syntax Guide

You need to know the exact syntax for writing rule expressions:

  • Range (numeric): >=0 And <=9999
  • Range (dates): >=#01/01/2020# And <=#31/12/2060# (Note the # hashes around dates)
  • Presence: Is Not Null (or set 'Required' to Yes)
  • Text list: "Active" Or "Maintenance" Or "Decommissioned"

Step-by-Step Blueprint

  1. CitizenID: Set Required to Yes. Set Input Mask to L\-00000;0;_.
    (L = one letter, \- = literal hyphen, 00000 = five digits. This handles Presence, Format, and Length!)
  2. EnergyCredits: Set Validation Rule to >=0 And <=9999. Set Validation Text to: "Energy Credits must be a value between 0 and 9999."
  3. DateOfBirth: Set Validation Rule to >=#01/01/1950# And <=Date().
    (Date() is Access's function for today's date).
  4. FineAmount: Set Default Value to 0. Set Validation Rule to >=0.
  5. Sector: Change Data Type to Lookup Wizard. Type the 5 zones (El Poblado, Laureles, Envigado, Belén, Centro). Most importantly, set Limit To List to Yes in Field Properties.

Test your gates: Switch to Datasheet view. Try to enter a negative FineAmount. Try to enter "Sabaneta" as a sector. The database should actively fight back!

📖 Digital Glossary

Validation
An automatic check carried out by the computer to ensure that data entered is reasonable, complete, and within acceptable boundaries.
Verification
A manual check to ensure data has been entered correctly and matches the original source (e.g., double data entry or visual checking).
Presence Check
A rule ensuring that a field is not left empty (null).
Range Check
A rule ensuring a numeric or date value falls within an acceptable minimum and maximum.
Length Check
A rule ensuring the number of characters in a field is within specified limits.
Format Check
A rule ensuring data matches a specific pattern of characters (often enforced via an Input Mask).
Lookup Check
A rule ensuring a value entered matches an item in a predefined list.
Input Mask
A field property in Access that forces a specific character-by-character entry pattern, combining Format and Length checks.
