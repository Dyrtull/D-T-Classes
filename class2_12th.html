Class 1: Logical Shifts & Non-Numeric Data

Theory Concepts

Logical Binary Shifts

A logical binary shift is an operation that moves the bits of a binary number to the left or right. When a bit is shifted out of the register, it is lost. When a new space is created at the end, it is filled with a `0`. This is different from an arithmetic shift, which preserves the sign bit. Logical shifts are used in computer systems for multiplication and division. A left shift of $n$ bits is equivalent to multiplying the number by $2^n$, while a right shift of $n$ bits is equivalent to dividing the number by $2^n$.

Non-Numeric Data Representation

Non-numeric data like text, images, and sound must also be represented in binary for a computer to process it.

  • Text (ASCII, Unicode): Each character is assigned a unique binary code. ASCII (American Standard Code for Information Interchange) uses 7 bits to represent 128 characters, which is enough for the English alphabet and common symbols. Unicode is a more modern standard that uses more bits, allowing it to represent a much wider range of characters and languages from around the world.
  • Images (Resolution, Color Depth): An image is made up of a grid of tiny squares called pixels.
    • Resolution: This is the number of pixels in an image, typically measured as width x height (e.g., $1920 \times 1080$). A higher resolution means more pixels, which results in a sharper image but a larger file size.
    • Color Depth (Bit Depth): This is the number of bits used to represent the color of a single pixel. A color depth of 1 bit allows for 2 colors ($2^1$), 4 bits allows for 16 colors ($2^4$), and 24 bits allows for over 16 million colors ($2^{24}$). A higher color depth gives more vibrant colors but increases the file size.
  • Sound (Sample Rate, Sample Resolution): Sound is an analogue wave, so it must be converted into a digital format.
    • Sample Rate: This is the number of times per second the amplitude of the sound wave is measured. It's measured in Hertz (Hz). A higher sample rate captures more detail of the sound wave, leading to higher quality but a larger file size. For example, $22$ kHz means $22,000$ samples are taken per second.
    • Sample Resolution (Bit Depth): This is the number of bits used to store the amplitude of each sample. A higher sample resolution provides a more accurate representation of the sound wave, improving quality but increasing file size.

Examples & Interactive Tool: Logical Shift

Example: A logical left shift of the 8-bit binary number `00011000` (Denary 24) by 3 bits. The original bits are shifted 3 places to the left. The 3 bits at the start are lost, and 3 zeros are added to the end.
`00011000` $\rightarrow$ `00110000` $\rightarrow$ `01100000` $\rightarrow$ `11000000`
The result is `11000000`, which is Denary 192.

Class 2: File Size, Compression & Data Transmission

Theory Concepts

Data Storage Units & File Size Calculation

Data storage units are based on powers of 2.

  • 1 byte = 8 bits
  • 1 KiB (Kibibyte) = $2^{10}$ bytes = 1,024 bytes
  • 1 MiB (Mebibyte) = $2^{10}$ KiB = 1,024 KiB
  • 1 GiB (Gibibyte) = $2^{10}$ MiB = 1,024 MiB
To calculate the file size of an uncompressed image:
File Size (bits) = (Resolution Width $\times$ Resolution Height) $\times$ Color Depth (bits)

Compression: Lossy vs. Lossless

  • Lossless Compression: This method reduces file size without losing any data. The original data can be perfectly reconstructed from the compressed data. It's used for files where data integrity is critical, such as text documents or code. A common technique is Run-Length Encoding (RLE), which replaces sequences of identical data with a count and the data itself (e.g., `AAAA` becomes `4A`).
  • Lossy Compression: This method reduces file size by permanently removing some data that is considered less important to the human eye or ear. The original file cannot be restored exactly. It's used for media files like images (JPEG) and sound (MP3) where a small loss in quality is acceptable for a significant reduction in file size.

Data Transmission: Packets

When data is sent over a network, it is broken down into small units called packets. Each packet travels independently to the destination, where they are reassembled in the correct order. Each packet has a specific structure:

  • Header: Contains control information such as the source and destination IP addresses, packet sequence number, and protocol information.
  • Payload: This is the actual data being sent.
  • Trailer: Contains information to check for errors in transmission, such as a checksum.
