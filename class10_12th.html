Step-by-Step Planning Process

STEP 1: Introduce Your Project (5 min)

Start a new conversation with your chosen resource. Clearly define your starting point using a format similar to the following template:

  • Role: I am a student learning electronics and programming for the first time, and I have NO prior experience.
  • Goal: I need to design a project.
  • Project Name: [WRITE YOUR PROJECT NAME]
  • Function: [DESCRIBE WHAT YOU WANT IT TO DO]
  • Instruction: I need you to teach me as we proceed, explaining all concepts simply. Can you help me understand what components I will need?
STEP 2: Learn About Key Components (15 min)

Ask these questions sequentially, taking notes on the answers and following up on anything you do not understand.

  1. "What sensors would I need for this project? Explain each one simply."
    Action: Note what each sensor measures.
  2. "What microcontroller should I use—Arduino or ESP32? Why?"
    Action: Understand the fundamental difference and note the recommended option.
  3. "What outputs will I need (LEDs, buzzers, motors, screens, etc.)?"
    Action: List each output device and its function.
  4. "Are there any other components I'll need?"
    Action: List ancillary items like resistors, wires, and breadboards.

Deliverable: Write down each component in your notebook with a simple description of its role.

STEP 3: Understand the Project Logic (15 min)

Ask the resource to explain the functional sequence of your project:

  1. "Can you explain in simple steps how this project would work? Outline the order: First this happens, then the device does this, and so on."
    Action: Document the resulting PROJECT LOGIC sequence in your notebook.
  2. "What is a 'threshold' and how would I decide what specific number to use in the code?"

Keep asking "why" and "how" questions until you grasp the basic operational flow.

STEP 5: Ask About Programming (10 min)

You will not write full code today, but you need to prepare for the programming stage:

  1. "What programming concepts will I need to learn for this project? Please list and describe them simply."
    Action: List concepts such as variables, if statements, loops, functions, and libraries.
  2. "Can you show me what a tiny piece of the code might look like? Specifically, the part that reads the sensor."
    Action: Copy this snippet into your notebook.
STEP 6: Identify Challenges (10 min)

Be honest about your concerns and identify potential difficulties:

  1. "What parts of this project might be challenging for a beginner? What should I be careful about?"
    Action: Write down challenges (e.g., sensor calibration, secure connections, waterproofing, power management).
  2. "How can I make this project simpler or break it into smaller steps?"
    Action: Note the simplest viable version of your project.
STEP 7: Final Component List (5 min)

Ask the resource: "Can you provide a final shopping list of exactly what components I'll need, with quantities?"

Best Practices

Best Practices (DO)

  • Ask "why" and "how" constantly.
  • Request explanations in simpler terms if confused.
  • Ask for examples or step-by-step breakdowns.
  • Tell the resource when you do not understand a concept.
  • Request it to "explain like I'm a complete beginner."

Pitfalls to Avoid (DON'T)

  • Copy information you do not fully understand.
  • Pretend you know something when you don't.
  • Skip over confusing parts or rush through the process.
  • Be afraid to ask clarifying questions.

Great Follow-Up Questions

  • "Can you explain that with a real-world analogy?"
  • "What does [technical term] mean in simple words?"
  • "Can you break that into smaller steps?"
  • "What's the most common beginner mistake with this?"
  • "What alternatives could I use instead of [component]?"
