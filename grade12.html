Week 1: ICT (Databases) & Computer Science (Data Representation)

ICT Section: Databases - Part 1

Documents for class Forms computers

Goals:

Cognitive Goals: Understand the characteristics of flat-file and relational databases. Learn how to create a database structure, including fields, data types, and keys. Learn to manipulate data using queries, sorting, and searching.

SEL Goals: Develop logical thinking and an appreciation for organized data.

Theory Concepts:

Understand the characteristics, uses, advantages, and disadvantages of flat-file and relational databases. Learn how to create a database structure, including setting field names, data types, and primary keys.

Definitions:

Database, Flat File Database, Relational Database, Primary Key, Foreign Key, Data Types, Query, Sort Data, Search and Select Data.

Examples:
Data Types:

A `Price` field should be `Currency`, a `Date of Birth` field should be `Date/Time`.

Primary Key:

In a `Students` table, `StudentID` would be a good primary key because each student has a unique ID number.

Query:

To find all customers who live in 'London' and have a balance greater than $1000, you would use a query with the criteria `City = 'London' AND Balance > 1000`.

In-Class Exercises:
Database Structure Design:

Design a `Books` table for a school library. Suggest three suitable field names and their data types.

Data Import and Manipulation:

Import data from a `Students.csv` file, set `StudentID` as the primary key, sort by `LastName`, and create queries to find students with a 'A' or 'B' grade, or a 'C' grade born after 2005.

Possible Homework:
Validation Rules:

For the `Books` table you designed, suggest one validation rule for each of your three fields.

Database Types:

Write a short paragraph explaining the difference between a flat-file database and a relational database, including an advantage and a disadvantage of each type.

Computer Science Section: Data Representation

Document 1 for class Document 2 for class

Goals:

Cognitive Goals: Review number systems (denary, binary, hexadecimal) and their conversions. Understand the role of a bit and the use of hexadecimal as a shorthand.

SEL Goals: Develop problem-solving skills through conversion drills and applying knowledge to real-world contexts like MAC addresses and error codes.

Theory Concepts:
Class 1: Number Systems:

Review the foundational concept of `place value` for denary, binary, and hexadecimal number systems. Explain the role of a single `bit`. Discuss the benefits of using `hexadecimal`.

Class 2: Binary Arithmetic & Two's Complement:

Provide a detailed explanation of the rules for `binary addition` and define `overflow`. Explain the step-by-step method for representing negative numbers using `two's complement`.

Examples:
Denary to Binary:

Convert the denary number 197 to its 16-bit binary equivalent, showing the place value calculation.

Binary to Hexadecimal:

Convert the 8-bit binary number `11010110` to hexadecimal by splitting it into two nibbles.

Binary Addition with Overflow:

Add the 8-bit binary numbers `10110011` and `01001100`. The sum will result in a 9-bit number, demonstrating an overflow in an 8-bit register.

Two's Complement Conversion:

Convert the denary number -85 to its 8-bit two's complement representation, showing each step.

In-Class Exercises:
Class 1: Conversion Drills:

  • - Convert Denary numbers 123, 89, and 255 to 8-bit binary.
  • - Convert the binary numbers 00111100, 11100011, and 01010101 to hexadecimal.
  • - Convert hexadecimal numbers 3A, F0, and 1B to denary.
  • - Convert an 8-bit binary number to hexadecimal, then convert the hexadecimal result back to denary to verify the original binary's value.

Class 2: Addition & Two's Complement Practice:

**Addition Drills:** Perform the following 8-bit binary additions, identifying if an overflow occurs for each: `01011010 + 00100111`, `11001100 + 01010101`, `01111111 + 00000010`.
**Two's Complement Conversions:** Convert the denary numbers -56, 105, and -128 to 8-bit two's complement. Then, convert the 8-bit two's complement numbers `11110011` and `01101010` back to denary.

Possible Homework:
Class 1:

Research and write a short paragraph explaining how hexadecimal is used in a specific computer science context.

Class 2:

A computer system uses 8-bit two's complement to represent signed integers. Perform the following calculation: `50 + (-30)`. You must show the conversion, perform the binary addition, and then convert the result back to denary.

Computer Science Section: Data Representation & Data Storage

Link to Class Presentation

Goals:

Cognitive Goals: Explain logical binary shifts and how non-numeric data (text, images, sound) is represented. Understand data storage units, file size calculation, lossy and lossless compression, and the structure of data packets.

SEL Goals: Foster an appreciation for how computers handle different types of data and the trade-offs involved, such as file size and quality.

Theory Concepts:
Class 1: Logical Shifts & Non-Numeric Data:

Explain `logical binary shifts`. Explain how `non-numeric data` is represented: Text (ASCII, Unicode), Images (resolution, color depth), and Sound (sample rate, sample resolution).

Class 2: File Size, Compression & Data Transmission:

Review `data storage units` and `file size calculation`. Define `lossy` and `lossless compression`. Introduce `data packets` and the structure of a packet header, payload, and trailer.

Examples:
Logical Shift:

Demonstrate a logical left shift of the 8-bit binary number `00011000` (Denary 24) by 3 bits. The result is `11000000` (Denary 192).

Image Representation:

Show how a simple image with a resolution of 2x2 pixels can be represented with a 1-bit color depth (black and white) versus a 4-bit color depth (16 colors). Explain the resulting difference in file size.

Compression:

Apply a simple Run-Length Encoding (RLE) lossless compression to the string `AAAABBBCCDAAA`. The result is `4A3B2C1D3A`.

In-Class Exercises:
Class 1: Shift Drills & File Size Impact:

**Shift Drills:** Perform the following logical shifts and state the new binary and denary values (assuming 8-bit registers):
- Logical left shift of `00101010` by 2 bits.
- Logical right shift of `10100000` by 3 bits.
**File Size Impact:** An audio file is recorded with a sample rate of 22 kHz and a sample resolution of 8 bits. Describe what happens to the file size and quality if: a) the sample rate is doubled, and b) the sample resolution is halved.

Class 2: File Size, Compression & Data Transmission:

**File Size Problem:** A picture has a resolution of 1920 x 1080 pixels with a color depth of 24 bits. Calculate its file size in Gibibytes (GiB) and explain your steps.
**Compression Application:** You are sending a text document and a family photograph. For each, which type of compression (lossy or lossless) would you use and why?

Possible Homework:
Class 1:

**RGB and Binary:** A graphics card uses hexadecimal to represent colors in RGB format. The hex code for a specific shade of blue is `#0000FF`. Research this format and break down the hex code into its red, green, and blue components. Then, convert each component into an 8-bit binary number.

Class 2:

**Compression Research:** Research one specific lossy compression algorithm (e.g., how JPEG works by averaging colors) and one specific lossless compression algorithm (e.g., Huffman coding). Explain the basic principle of how each reduces file size.

Computer Science Section: Data Transmission

Link to Class Presentation

Goals:

Cognitive Goals: Understand data transmission modes (simplex, half-duplex, full-duplex) and error detection methods. Explain how a CPU's performance is affected by cores, cache, and clock speed. Differentiate between embedded systems and general-purpose computers, and between system and application software.

SEL Goals: Develop a methodical approach to problem-solving and debugging. Foster a deeper understanding of technology's real-world applications.

Theory Concepts:
Class 1: Data Transmission & Hardware:

Discuss the differences between simplex, half-duplex, and full-duplex data transmission modes, providing a real-world example for each. Explain the need for error detection and describe two methods: parity check (odd and even) and checksum. Introduce the core components of the CPU in a Von Neumann architecture and walk through the Fetch-Decode-Execute (FDE) cycle.

Class 2: Hardware & Software:

Explain how a CPU's performance is affected by its number of cores, the size of its cache, and its clock speed. Define an embedded system and distinguish it from a general-purpose computer. Differentiate between system software and application software. List and describe the basic functions of an operating system (OS).

Examples:
Transmission Modes:

A radio broadcast is simplex, a walkie-talkie is half-duplex, and a telephone call is full-duplex.

Parity Check:

For the 8-bit data `10110010`, explain that with even parity, the parity bit is 1, and with odd parity, it is 0.

CPU Performance:

A quad-core CPU at 3.0 GHz will generally handle multitasking better than a dual-core CPU at 3.5 GHz. A larger cache reduces the need to access slower main memory (RAM).

Embedded vs. General-Purpose:

A car's engine management system is an embedded system; a laptop is a general-purpose computer.

In-Class Exercises:
Class 1: Data Transmission & FDE Cycle:

**Parity Check:** A block of data is transmitted with an odd parity check for each byte. The received bytes are: `10101011` and `01100100`. Identify which byte contains an error and explain why.
**FDE Cycle:** Describe, in detail, the steps of the Fetch-Decode-Execute cycle for a CPU to carry out a simple instruction. Ensure you mention at least four different registers and their purpose in the process.

Class 2: System Identification & OS Functions:

**System Identification:** Given a device like a digital camera, a washing machine, and a gaming console, determine if each is an embedded system or a general-purpose computer and provide a justification.
**OS Functions:** List and describe three essential functions of an operating system. For each function, provide a specific example of when it would be used.

Possible Homework:
Class 1:

Research the algorithm for a simple check digit, such as the one used in the last digit of an International Standard Book Number (ISBN). Apply the algorithm to a hypothetical 9-digit number and determine the correct check digit.

Class 2:

Research the purpose of a device driver. Explain how it is part of the operating system's role in managing peripherals and why a new device might not work correctly without a driver.

Computer Science Section: Programming Concepts & Algorithms

Link to Class Presentation
PDFs repository

Goals:

Cognitive Goals: Differentiate between compilers and interpreters. Introduce fundamental programming concepts like variables, constants, data types, and control structures (sequence, selection, iteration). Learn about standard algorithms (e.g., linear search, bubble sort).

SEL Goals: Build logical reasoning skills and attention to detail by tracing algorithms and writing simple pseudocode.

Theory Concepts:
Class 1: Data Storage, Translators & Programming Introduction:

Review primary vs. secondary storage. Introduce virtual memory as a technique to compensate for low RAM. Explain the difference between compilers and interpreters. Introduce fundamental programming concepts: variables, constants, and the five basic data types.

Class 2: Programming Flow & Standard Algorithms:

Explain the fundamental control structures: sequence, selection (`IF...THEN...ELSE`), and iteration (e.g., `FOR...NEXT`). Introduce the three types of loops: count-controlled, pre-condition, and post-condition. Introduce standard methods of solution: linear search, bubble sort, totalling, counting, and finding maximum/minimum/average values.

Examples:
Virtual Memory:

The operating system uses virtual memory by swapping unused "pages" of data from RAM to the hard drive, freeing up space for active programs.

Compilers vs. Interpreters:

A C++ program is compiled once into an executable file, and the compiler lists all errors. A Python program is interpreted line by line, and the interpreter stops as soon as it finds an error.

Control Structures:

Write pseudocode for an `IF` statement to check if a number is positive. Write pseudocode for a `FOR` loop to count from 1 to 10.

Linear Search:

A linear search on the list `[4, 2, 8, 1, 9]` for the number 8 checks each element one by one.

In-Class Exercises:
Class 1: Storage & Pseudocode Practice:

**Storage Scenario:** A user wants to run a large video editing application but only has 4GB of RAM. The OS uses virtual memory. Explain what virtual memory is and how it helps the user in this scenario.
**Pseudocode Practice:** Write pseudocode to declare variables for a user's name, age, and a constant for the current year. Assign values to each variable and output them.

Class 2: Loop & Algorithm Tracing:

**Loop and Condition:** Write a pseudocode algorithm that asks the user to enter numbers until they enter -1. It should then output the total sum of all the positive numbers entered.
**Algorithm Tracing:** Manually trace a linear search for the number 7 on the list `[10, 5, 2, 8, 7]`, noting the comparisons made at each step.

Possible Homework:
Class 1:

Write a complete pseudocode program that prompts the user to enter a username (string) and a password (string). The program should then check if the username is "admin" and the password is "password123". If both are correct, it should output "Access Granted."

Class 2:

Write the complete pseudocode for a bubble sort algorithm that sorts a 1D array of 5 integers in ascending order. You do not need to read the numbers from a user; you can assume the array is already populated.

Week 5: ICT (Theory Review) & Computer Science (Hardware & Software)

ICT Section: Theory Review - Systems, Devices, and Storage

Link to Class Presentation

Goals:

Cognitive Goals: Review computer systems, hardware, software, and storage. Understand the differences between ROM/RAM and backing storage/internal memory. Review networks and security concepts.

SEL Goals: Develop self-assessment skills by identifying their own strengths and weaknesses in theory topics.

Theory Concepts:

Review and understand the different types and components of computer systems, hardware, software, and storage. Review key concepts in networks and security.

Definitions:

Hardware, Software, System Software, Application Software, ROM, RAM, Backing Storage, Input/Output/Storage Devices, Network Types, Router, Anti-malware software.

Examples:
Hardware:

A CPU, motherboard, or a keyboard.

Software:

An operating system like Windows is system software. A word processor like Microsoft Word is application software.

Storage:

An external hard drive is backing storage. RAM is internal memory.

Input/Output:

A barcode reader is an input device. A 3D printer is an output device.

Networks:

A Wi-Fi network in a house is a LAN. The internet is a WAN.

In-Class Exercises:
Hardware vs. Software:

Classify the following as either Hardware or Software: a) A mouse b) A database program c) A scanner d) A device driver.

Storage Comparison:

Create a table to compare RAM and ROM based on three characteristics.

Device Identification:

For each of the following scenarios, suggest an appropriate input device and output device and briefly explain why it is suitable: a) A supermarket self-checkout machine. b) A system for playing a video game on a computer.

Network Security:

A user receives an email with an attachment from an unknown sender. a) Explain one security threat associated with this scenario. b) Describe one method of protection that could prevent this threat.

Possible Homework:
Storage Technologies:

Describe one advantage and one disadvantage of magnetic, optical, and solid-state storage.

Computer Types:

List two characteristics of a mobile computer and two characteristics of a desktop computer.

Computer Science Section: Algorithms & Programming

Link to Class Presentation

Goals:

Cognitive Goals: Explain the program development life cycle. Differentiate between various validation and verification checks. Understand the use of trace tables, procedures, functions, and the difference between local and global variables.

SEL Goals: Encourage a systematic and methodical approach to problem-solving and debugging. Foster independence by writing and debugging their own functions.

Theory Concepts:
Class 1: Algorithm Design & Validation:

Explain the four stages of the `program development life cycle`. Provide specific, challenging examples for each type of `validation check`. Explain the difference between `validation` and `verification`.

Class 2: Tracing, Debugging & Functions:

Explain the purpose and construction of a `trace table`. Define the role of `procedures` and `functions`. Explain `parameters` and the difference between `local` and `global` variables.

Examples:
Validation:

A `format check` for an email address (e.g., checking for the '@' symbol and a domain). A `check digit` on a product barcode.

Verification:

A `double-entry check` for a password or a financial transaction. A `visual check` of data entered into a spreadsheet.

Functions:

Write a function in pseudocode to calculate the area of a rectangle, taking `Length` and `Width` as parameters and returning the result.

Variables:

Show an example where a global variable is changed inside a procedure, and contrast this with a local variable that is only accessible within the procedure.

In-Class Exercises:
Class 1: Form Design & Distinguishing Checks:

**Form Design:** Design a series of validation checks for a student registration form, with fields for `StudentID`, `Email`, and `Age`.
**Distinguishing Checks:** Explain the difference between validation and verification using a specific example for each.

Class 2: Debugging & Functions:

**Algorithm Debugging:** A pseudocode algorithm is designed to find the average of five numbers but has a logic error. Students will be given the code, trace it with a trace table, identify the error, and suggest a correction.
**Function Writing:** Write a function called `IsPositive` that takes an integer as a parameter and returns the Boolean value `TRUE` if the number is greater than zero, and `FALSE` otherwise.

Possible Homework:
Class 1:

Find a real-world registration form and list all the validation and verification checks you think it would need.

Class 2:

Write a function called `FahrenheitToCelsius` that takes a temperature in Fahrenheit as a parameter and returns the value converted to Celsius.
